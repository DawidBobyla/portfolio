Case Study: Inventory & Warehouse System

The game required multiple storage types (pawnshop and player house), each implemented differently across the project.

Challenge:
Unifying access to inventory data without tightly coupling systems.

Solution:

  • Base AWarehouse class with derived implementations
  • UInventoryComponent attached to derived warehouses for item data
  • IInventoryInterface for consistent access

Key Insight:
Used composition for data and inheritance for behavior, combined with interface-based access.

Result:
Reusable, scalable system used across multiple gameplay features.

Case Study: Multiplayer & UI Optimization

Scaling the game to multiple locations exposed performance issues in network replication.

Problem:
Each PriceTag actor replicated independently, causing high network load and performance drops when multiple pawnshops were introduced.

Solution:

  • Removed replication from individual PriceTag actors
  • Introduced central actor replicating array of prices
  • Updated UI client-side based on replicated data
  • Disabled world-space UI updates when player was far away (tick optimization)

Result:
Significantly reduced network traffic and eliminated performance issues when scaling the number of objects.

Code Examples

Automated Build Execution

Core method used to build the game using Unity BuildPipeline:

private static void BuildGame(BuildOptions buildOptions, BuildType buildType, bool buildAddressables = true)
{
    if (buildAddressables)
    {
        BuildAddressables();
    }

    string fullPath =
        $"{GetArgBuildPath()}/{Application.productName}/{GetBuildVersion()}/{buildType}/";

    Directory.CreateDirectory(fullPath);

    string[] scenes = EditorBuildSettingsScene.GetActiveSceneList(EditorBuildSettings.scenes);

    BuildPlayerOptions buildPlayerOptions = new BuildPlayerOptions
    {
        scenes = scenes,
        locationPathName = fullPath + $"{Application.productName}.exe",
        target = EditorUserBuildSettings.activeBuildTarget,
        options = buildOptions
    };

    BuildReport buildReport = BuildPipeline.BuildPlayer(buildPlayerOptions);

    if (buildReport.summary.result != BuildResult.Succeeded)
    {
        Directory.Delete($"{GetArgBuildPath()}/{Application.productName}/{GetBuildVersion()}/", true);
        throw new Exception();
    }
}

Handles full build execution, including addressables, output paths, and failure cleanup.

Command-Line Argument Parsing

Used to control build behavior from CI/CD pipelines:

private static string GetArg(string name)
{
    var args = Environment.GetCommandLineArgs();

    for (int i = 0; i < args.Length; i++)
    {
        if (args[i] == name && args.Length > i + 1)
        {
            return args[i + 1];
        }
    }

    return null;
}

Allows passing build parameters such as build path, type, and suffix from CI tools.

Build Versioning (Git Integration)

Build version generated from Git commit date and commit hash:

public static string GetBuildVersion()
{
    string output = RunGitCommand("log -1 --format=%cd");

    if (DateTime.TryParseExact(output, "ddd MMM d HH:mm:ss yyyy zzz",
        CultureInfo.InvariantCulture, DateTimeStyles.None, out DateTime lastCommitDate))
    {
        string buildVersion = lastCommitDate.ToString("yyyy.MM.dd_HH.mm") +
                              $"_{GetLastCommitIdShort()}{GetArgSuffix()}";
        return buildVersion;
    }
    else
    {
        string buildVersion = DateTime.Now.ToString("yyyy.MM.dd_HH.mm") +
                              $"_{GetLastCommitIdShort()}{GetArgSuffix()}";
        return buildVersion;
    }
}

Ensures traceable builds using Git commit metadata.

CI Error Reporting (JUnit)

Custom JUnit report generation for CI pipelines:

public static void CreateReport(List<string> errors, string reportName, string duration)
{
    if (errors.Count > 0)
    {
        for (int i = 0; i < errors.Count; i++)
        {
            CreateReport(errors[i], $"{reportName} Case {i + 1}", duration);
        }
    }
    else
    {
        CreateReport("", $"{reportName} Case {1}", duration);
    }
}

Enables CI systems (Jenkins / GitLab) to display structured build errors.
